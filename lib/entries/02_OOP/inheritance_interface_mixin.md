## 상속 · 인터페이스 · mixin 비교

| 구분         | 상속 (`extends`)                 | 인터페이스 (`implements`)                     | mixin (`with`)                           |
|--------------|----------------------------------|-----------------------------------------------|------------------------------------------|
| 목적         | 코드 재사용 + 확장                | 계약(규약) 강제, 다형성 제공                  | 공통 기능 조각을 섞어 넣기                |
| 다중 사용    | 불가능 (단일 상속만 가능)         | 가능 (여러 개 구현 가능)                      | 가능 (여러 개 mixin 조합 가능)            |
| 부모 메서드  | 그대로 사용 + override 가능       | 반드시 override 필요                          | 그대로 사용 가능, 필요시 override 가능     |
| 관계 표현    | is-a (자식은 부모의 일종이다)    | is-a-capable-of (이 능력을 반드시 가져야 한다) | has-capability-of (이 기능을 가질 수 있다) |
